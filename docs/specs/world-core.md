# World Core 受入仕様（計画段階）

この仕様はLISS-0018およびLISS-0019〜LISS-0024のPhase 1テスト設計の基準とする。今回の作業ではテストを実行しない。

## Scenario 1: プレイヤーのログイン

Given 空のワールドと一意なプレイヤーIDがある
When ログインイベントをワールドへ投入する
Then プレイヤーが一度だけ登録される
And 同じイベントを再処理しても重複状態にならない

## Scenario 2: 3次元距離による対象判定

Given 実行者と対象物が既知の3次元座標にある
When 魔法イベントを解決する
Then ユークリッド距離がコマンドの射程以内の場合だけ対象効果が発生する
And 射程外では対象物と環境エーテルに副作用を発生させない

## Scenario 3: 環境エーテルによる魔法威力

Given 同じ魔法、同じ実行者、同じ対象物がある
And 発動地点の属性エーテル状態が異なる
When それぞれの魔法イベントを実行する
Then 環境状態に応じて実効威力が異なる
And 同じワールド状態から同じイベント列を実行した結果は一致する

## Scenario 4: エーテル注入と周辺伝播

Given 発動地点とその周囲のEtherFieldがある
When 属性魔法が対象物へ命中する
Then 対象物への直接効果と環境エーテル注入が別の結果として記録される
And 設定された伝導率に従って周辺セルへ影響が伝播する

## Scenario 5: エーテル減衰

Given 属性エーテルが基準値より高いセルがある
When 時間が経過する
Then その属性値は設定された減衰率に従って基準値へ近づく
And 下限値を下回らない

## Scenario 6: 伝導率の変化

Given 属性エーテルが基準値から高い方向または低い方向へ離れている
When 伝導率を計算する
Then 基準値との差が大きいほど当該属性の伝導率が増加する
And 伝導率は設定された上限・下限内に収まる

## Scenario 7: 属性相互作用による二次影響

Given 複数属性のエーテルが相互作用し、バランス条件を超えている
When 環境更新を実行する
Then定義された二次イベントが発生する
And 二次イベントは元イベントの時刻と連番から再現できる

## Scenario 8: 安全なコピーと更新

Given Playerをコンテナへ登録し、同じIDの状態を更新する
When コピー代入またはコンテナ更新が発生する
Then所有状態が共有されず、破棄順に依存した動作にならない
And更新元と登録済み状態の変更範囲が仕様どおりになる

## Scenario 9: 空キューと欠損対象

Given イベントキューが空、または対象IDが存在しない
When ワールドステップを処理する
Thenクラッシュせず、明示的な拒否または無効イベントとして記録する

## Scenario 10: ランタイムの開始と停止

Given World runtimeが停止している
When 開始・停止を複数回要求する
Thenワーカーが重複起動せず、停止要求後に処理が収束する
And同じ入力列を固定ステップで処理できる

## Scenario 11: 20 FPSフレーム境界でのActionQueue

Given サーバーが20 FPS（1フレーム50ms）の固定ステップで動作している
And複数ユーザーのActionが同一フレームの入力受付期間に到着する
When サーバーが次のフレーム処理を開始する
Thenフレーム境界までに到着したActionをすべてActionQueueへ確定する
And確定したActionを到着順に一度ずつ処理する
Andフレーム処理中に到着したActionは次のフレームへ繰り越す
And同じ入力列を同じ到着順で再生した結果は一致する
