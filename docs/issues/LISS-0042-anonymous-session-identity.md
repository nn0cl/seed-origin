# LISS-0042: 認証なし仮セッションと内部ID

- Status: review
- Phase: phase-1-red
- Priority: high
- Parent: LISS-0041
- Related: LISS-0051

## Scope

認証なしで接続を受け付け、サーバー側で衝突しない内部IDを自動採番する。申告ユーザーIDは認証情報ではなく、過去セッションとの名寄せキーとして扱う。

## Acceptance criteria

- 内部IDはクライアント入力を採用せずサーバーが採番する。
- 同じ有効な申告IDは同じ名寄せIDへ結びつく。
- 未申告・不正形式・長すぎるIDは匿名セッションとして扱うか拒否する契約を固定する。
- セッション切断で内部IDを再利用しない。
- 認証済みユーザーと誤認させるAPI名・状態を使用しない。

実装スライス: LISS-0053。永続化と通信接続は後続Issueで扱う。
